# Задание 1. Шаблоны

## Задача 1. Кэшируемая дробь

### Текст задачи
В класс **Дробь** необходимо добавить интерфейс с двумя основными действиями:
- получение вещественного значения;
- установка числителя и знаменателя.

Также требуется создать такую версию дроби, которая **кэширует вычисление вещественного значения**.  
Если ранее сущность `Fraction` в проекте отсутствовала, нужно реализовать ее со следующими особенностями:
- дробь имеет числитель и знаменатель;
- создается по числителю и знаменателю;
- может возвращать строку вида `числитель/знаменатель`;
- отрицательные значения должны обрабатываться корректно;
- знаменатель не может быть отрицательным;
- сравнение дробей должно выполняться по состоянию.

### Алгоритм решения
1. Создан класс `Fraction` с полями `numerator` и `denominator`.
2. Реализован интерфейс для:
   - получения вещественного значения;
   - изменения числителя;
   - изменения знаменателя.
3. Создана расширенная версия дроби с кэшированием результата вычисления `doubleValue()`.
4. При первом вычислении вещественного значения результат сохраняется.
5. При повторных вызовах возвращается уже сохраненное значение без повторного деления.
6. При изменении числителя или знаменателя кэш сбрасывается.
7. В `main` демонстрируется:
   - создание дроби;
   - получение вещественного значения;
   - повторный вызов метода;
   - обновление числителя/знаменателя и пересчет результата.

# Задание 2. Структурные шаблоны

## Задача 1. Подсчет количества мяуканий

### Текст задачи
Необходимо воспользоваться классом **Кот** и методом, который принимает набор объектов, способных мяукать.  
Требуется передать кота в этот метод так, чтобы **после завершения работы можно было узнать, сколько раз кот мяукал**, при этом **сам класс кота изменять нельзя**.

Если ранее класса `Cat` не было, нужно создать:
- сущность `Cat` с именем;
- метод `meow()`, выводящий строку вида `Имя: мяу!`;
- интерфейс для всех объектов, которые умеют мяукать.

### Алгоритм решения
1. Создан интерфейс `Meowable` с методом `meow()`.
2. Реализован класс `Cat`, который умеет мяукать.
3. Создан метод, принимающий набор объектов `Meowable` и вызывающий `meow()` у каждого.
4. Для подсчета количества мяуканий использован структурный шаблон-обертка:
   - создается объект, который содержит ссылку на настоящего кота;
   - каждый вызов `meow()` делегируется исходному объекту;
   - одновременно увеличивается счетчик вызовов.
5. После завершения работы метода счетчик выводится на экран.
6. В `main` демонстрируется передача кота в общий метод и последующий вывод количества мяуканий.

# Задание 3. Список

## Задача 1. Удаление всех элементов E из списка L

### Текст задачи
Составить программу, которая **удаляет из списка `L` все элементы `E`**, если такие элементы присутствуют.

### Алгоритм решения
1. Пользователь вводит список элементов и значение `E`, которое необходимо удалить.
2. Создается новый список либо выполняется удаление из существующего списка.
3. Все элементы списка последовательно проверяются.
4. Если элемент равен `E`, он удаляется.
5. После завершения обработки выводится итоговый список без удаленных значений.
6. В `main` демонстрируется работа алгоритма на нескольких примерах.

# Задание 4. Map

## Задача 1. Формирование уникальных логинов по фамилии

### Текст задачи
На вход подаются фамилии и имена учеников.  
Необходимо сформировать и вывести **уникальный логин** для каждого ученика по следующему правилу:
- если фамилия встретилась впервые, логин равен фамилии;
- если фамилия уже встречалась, к ней добавляется номер появления: `2`, `3` и так далее.

Пример:
- `Иванова Мария` → `Иванова`
- `Петров Сергей` → `Петров`
- `Петров Иван` → `Петров2`

### Алгоритм решения
1. Используется структура `Map<String, Integer>`, где:
   - ключ — фамилия;
   - значение — количество уже встреченных учеников с этой фамилией.
2. Для каждой новой записи программа проверяет, встречалась ли фамилия раньше.
3. Если нет — фамилия выводится без изменений и добавляется в `Map`.
4. Если да — увеличивается счетчик, и к фамилии добавляется соответствующий номер.
5. В результате для каждого ученика формируется уникальный логин.
6. В `main` показывается чтение входных данных и печать логинов.

# Задание 5. Set

## Задача 1. Звонкие согласные, встречающиеся хотя бы в одном слове

### Текст задачи
Файл содержит текст на русском языке.  
Необходимо **напечатать в алфавитном порядке все звонкие согласные буквы**, которые входят **хотя бы в одно слово** текста.

### Алгоритм решения
1. Из файла считывается текст.
2. Текст разбивается на слова.
3. Для каждого слова анализируются символы.
4. Проверяется принадлежность символа множеству звонких согласных русского алфавита.
5. Подходящие буквы добавляются в `Set`, чтобы исключить повторы.
6. После обработки всего текста множество сортируется в алфавитном порядке.
7. Итоговый набор букв выводится на экран.

# Задание 6. Очередь

## Задача 5. Переписать элементы из очереди L1 в очередь L2 в обратном порядке

### Текст задачи
Необходимо переписать элементы из очереди `L1` в очередь `L2` **в обратном порядке**.

### Алгоритм решения
1. Исходная очередь `L1` заполняется элементами.
2. Для обращения порядка используется вспомогательная структура:
   - стек;
   - список;
   - либо иная промежуточная коллекция.
3. Все элементы извлекаются из `L1` и помещаются во временное хранилище.
4. Затем элементы заносятся в `L2` в обратной последовательности.
5. В результате `L2` содержит те же элементы, но в обратном порядке.
6. В `main` выводятся исходная и результирующая очереди.

# Задание 7. Stream

## Задача 1. Построение ломаной из точек через Stream API

### Текст задачи
Дан набор объектов типа `Point`.  
Необходимо с помощью стрима:
- взять все точки;
- убрать точки с одинаковыми координатами `X, Y`;
- отсортировать точки по `X`;
- у отрицательных значений `Y` сделать значения положительными;
- собрать результат в объект `Polyline`.

Если классы `Point`, `Line` и `Polyline` ранее отсутствовали, их нужно реализовать.

### Алгоритм решения
1. Создан класс `Point` с координатами `x` и `y`.
2. Создан класс `Polyline`, хранящий набор точек.
3. Исходный список точек обрабатывается через `Stream`.
4. На этапе обработки:
   - удаляются дубликаты по координатам;
   - выполняется сортировка по `x`;
   - отрицательные значения `y` заменяются на положительные.
5. Полученная последовательность собирается в объект `Polyline`.
6. В `main` демонстрируется исходный набор точек и результат работы стрима.

## Задача 2. Группировка имен людей по номеру из текстового файла

### Текст задачи
Дан текстовый файл со строками вида:
- `Вася:5`
- `Петя:3`
- `Аня:5`

Номера у людей могут повторяться, а у некоторых людей номера может не быть.  
Необходимо написать стрим, который:
- считывает всех людей из файла;
- приводит имена к виду: первая буква заглавная, остальные строчные;
- удаляет людей без номера;
- группирует имена по номеру.

Ожидаемый пример результата:
`[5:[Вася, Аня], 3:[Петя]]`

### Алгоритм решения
1. Из файла считываются все строки.
2. Каждая строка разбирается на имя и номер.
3. Некорректные записи и записи без номера отбрасываются.
4. Имена приводятся к нормализованному регистру.
5. С помощью `Stream API` выполняется группировка по номеру.
6. В качестве результата формируется структура `Map<Integer, List<String>>`.
7. В `main` выводится сгруппированный результат в удобном виде.
